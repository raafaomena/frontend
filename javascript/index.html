1. The Foundation: Integrating and Debugging JavaScript

[cite_start]

Before writing any code, it is crucial to understand the two most fundamental mechanics: how to correctly link JavaScript to a web page for optimal performance and how to use the browser's built-in tools to get feedback and troubleshoot issues[cite: 318, 1661].

1.1. Efficiently Adding JavaScript to a Web Page

There are two primary methods for including JavaScript in an HTML document. [cite_start]Both are strategically designed to be "non-blocking," ensuring the script runs only after the HTML is loaded and available for manipulation[cite: 1670].

  • Inline JavaScript: The code is placed directly within <script> tags, located just before the closing </body> tag of the HTML document.
  • External JavaScript Files: A <script> tag with a src attribute links to an external .js file. This tag should be placed in the <head> of the HTML document and must include the defer attribute.
[cite_start]

Using external files offers benefits such as the separation of concerns, easier maintenance, and performance boosts through browser caching[cite: 1673]. [cite_start]Crucially, both methods are non-blocking, ensuring the browser is not delayed and can continue visual rendering for a faster user experience[cite: 1670].

2. Storing Information: The Role of Variables

[cite_start]

A variable is a "container" or "storage box" for a value that may change each time code is run[cite: 665].

2.1. Declaring Variables: let vs. const

[cite_start]

Modern JavaScript uses two primary keywords for declarations[cite: 671]:

    [cite_start]
  • let: Used for values that are expected to change or be reassigned, such as game scores[cite: 675, 676].
    • [cite_start]
  • const: Used for values that should not change after initial assignment, like a date of birth[cite: 674]. This is the preferred default to prevent accidental modifications.
[cite_start]

The older var keyword is no longer recommended[cite: 677].

2.2. Professional Naming Conventions

[cite_start]

Variable names must be unique within their scope, cannot contain spaces or dashes, and are case-sensitive[cite: 678, 680, 682]. [cite_start]The professional standard is camelCase, where the first word is lowercase and subsequent words are capitalized (e.g., userFirstName)[cite: 685, 687].

3. Manipulating Core Data Types

[cite_start]

A variable's data type determines the operations possible on it[cite: 704, 708].

3.1. Working with Text (Strings)

[cite_start]

A string is a sequence of characters enclosed in quotes[cite: 1233, 1234]. Key principles include:

    [cite_start]
  • Zero-Based Indexing: Characters are accessed by position starting from 0 using square brackets (e.g., myString[0])[cite: 1248, 1249].
    • [cite_start]
  • Length Property: The .length property reveals the total number of characters[cite: 1256].
    • [cite_start]
  • Built-in Methods: Actions like .includes(), .trim(), and .slice() allow for querying and transforming text[cite: 1263, 1271, 1278].
    • [cite_start]
  • Immutability: String methods do not change the original string; they return a new one that must be reassigned to persist[cite: 1228].

3.2. Working with Numbers

[cite_start]

Numeric variables store integers or floating-point numbers without quotes[cite: 979, 981].

    [cite_start]
  • Arithmetic: Use +, -, *, and / for standard calculations[cite: 993, 996].
  • Operator Precedence: JavaScript follows standard arithmetic rules (division/multiplication before addition/subtraction). [cite_start]Use parentheses () to override this[cite: 999, 1002].
    • [cite_start]
  • Shortcuts: Increment (++) and decrement (--) operators efficiently add or subtract one[cite: 974, 976].

3.3. Other Essential Types: Booleans and Null

    [cite_start]
  • Boolean: Holds only true or false; often used for logical results[cite: 713, 716].

4. A Defining Feature: JavaScript's Dynamic Typing

[cite_start]

JavaScript is a "loosely-typed" or "dynamically-typed" language[cite: 658, 663].

4.1. Understanding a Loosely-Typed Language

You do not need to specify a data type during declaration; [cite_start]JavaScript infers it from the value[cite: 707, 708]. [cite_start]Furthermore, a variable's type can change if it is reassigned a different kind of value[cite: 708].

4.2. Inspecting Data Types with the typeof Operator

[cite_start]

The typeof operator returns a string indicating the current data type (e.g., "string", "number")[cite: 1245, 1246]. [cite_start]Notably, typeof null returns "object," which is a known error in the language[cite: 624].
